## Entire Dead Modules

These modules are fully scaffolded (types, implementation, barrel exports) but **no runtime code imports them** (only re-exported by src/index.ts).

### 1. src/mcp/ — MCP Protocol Client

| File | Dead Exports |
|---|---|
| McpClient.ts | McpClient |
| McpManager.ts | McpManager |
| types.ts | McpServerConfig, McpTool, McpResource, McpPrompt, McpRequest, McpResponse, McpToolCall, McpToolResult |
| index.ts | Barrel re-exports all of the above |

**Status:** Complete orphan. No imports in src/, tests/, or TUI runtime code — only re-exported by src/index.ts. The MCP (Model Context Protocol) client was likely planned for external tool-server integration but never connected.

**Files to remove:** All files in src/mcp/

## Recommendations

### High Priority — Remove Entirely Dead Modules

Removing the 2 dead modules eliminates ~600+ lines of untested, unreachable code.

```bash
# Safe to delete entirely:
rm -rf src/mcp/
rm -rf src/tasks/
```

Also remove the corresponding re-export lines from src/index.ts.

### Medium Priority — Prune Dead Exports

Remove the 12 groups of dead exports listed above. This tightens the public API surface and makes it clearer what is actually wired in vs. scaffolded.

### Low Priority — Clean Up Barrel Files

The index.ts barrel files re-export everything from their module, including dead code. After removing dead modules and exports, update each barrel to only re-export what is actually used externally.

### Do NOT Remove

The following modules look unused at first glance but are **actively used** — just wired in through createMiMoStack.ts or the TUI layer:

- src/approval/ — Used by EngineBridge.ts and createMiMoStack.ts
- src/memory/ — Used extensively by MiMoLoop.ts and createMiMoStack.ts
- src/session/ — Used by createMiMoStack.ts and EngineBridge.ts
- src/validators/ — Used by createMiMoStack.ts
- src/tools/filesystem/, src/tools/shell/, src/tools/git/, src/tools/web/, src/tools/todo/ — All registered in createMiMoStack.ts
- src/repair/ — Used by MiMoLoop.ts and createMiMoStack.ts
- src/orchestration/ — Used by PipelineBridge.ts
